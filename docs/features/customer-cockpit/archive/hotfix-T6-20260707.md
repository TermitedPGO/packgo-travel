## 1(P0)customerUnreadCount 500 — TiDB ORDER BY 關聯子查詢

根因(監工 prod 探針定位):`adminCustomers.ts` guest 臂的 `lastContactSql`(GREATEST 內含

姊妹 `guestList` 因為 select 了 `customerProfiles.id` + 同表達式才沒炸。

再 ORDER BY(TiDB 就重用投影值);`customerUnreadCount` 與 weeklyCanary 呼叫「同一支」,口徑與
guestList 逐字一致。`docs/agent/30-templates.md` 通用地雷加第四條:TiDB 關聯子查詢必須同時在
SELECT(本機無 DATABASE_URL 測不到 SQL 方言,靠 prod 探針 + 審查佐證)。
