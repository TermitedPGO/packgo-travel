## 1（P0)外寄信宣稱有附件卻沒帶 — 根因修 + 兩道防線

**先取證**:prod agentMessages 唯讀探針調出完單測試步驟 3/5/6 的草稿卡 context。三張寄出的
金門草稿(id 1620434「附在信裡」、1620436「附在這封信裡」、1620437「隨信附上訂金收據」)
`context.replyAttachments` 全空;對照 Napa 那張(先有草稿才出文件)則有掛。**根因 = A**:先出
文件時當下沒草稿可掛 → 另一輪叫草稿寄,新草稿沒繼承附件,寄的就是這張空草稿。

**根因修**:`produceFollowupDraftForProfile` 建新草稿前,若文案聲稱附上檔案,撈這位客人最近
30 分內產生的 generated PDF,只掛「最新一份」(一封信一份;E2E step3/5 相隔 10 分,每種各掛
會讓請款信同時附上報價 — adversarial review 抓到,已收斂成單一最新)。寫進 context.replyAttachments。

**兩道防線**:
- (a) `followupDraftHonesty` 新增 `claimed_attachment_missing`(純 code 零 LLM):文案聲稱附上但
  hasAttachment=false → 擋不落卡;`escalationBox` 送出前加同一道窄閘,信文說附上卻沒帶任何附件
  → 不寄,回可行動錯誤。中英措辭表已排除未來式(我會附上/I will attach)、否定(沒有附件)、
  無關詞(附近/附加/附註/attached-to X)。
- (b) `generate_customer_document` 工具描述 + 回傳補明正確順序:沒草稿時先產生回信草稿再由系統
  掛 PDF;檔案沒掛上前信裡不要寫「附上」。

adversarial review(fresh subagent):無 blocker;1 個低風險英文誤擋已收斂 + 2 個低風險註記已處理。

## 3（P2)承諾相對日期 dueDate=null

`resolveEventDate` 新增純 code 分支「N 天內/N 日內/N 天後/N 天后」= 今天 + N(美西曆日,上限 60)。
配套修 `stripDateModifierSuffix`:整串相對天數表達開頭早退,不再把尾端「以內/後/后」剝掉(舊行為
