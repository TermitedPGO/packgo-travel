日期:2026-07-04 · commit `00a710f`(已 rebase 上 main、已 push origin/main)· 執行:Opus 4.8 · 派工:監工 live 驗收

## 1. 監工抓到什麼

v787 上線後 live 驗收抓到 A2(客人列表「最後往來」口徑)兩個真缺陷:

- **P0 — customerList 註冊會員全消失**。`admin.customerList` 回空陣列(`includeHidden:true` 也空);0909(userId 60001)在 v786 還在列表。
- **P1 — guestList 的 lastContactAt 實際回 `updatedAt`**。Emerald(profileId 2760016)真 `lastInboundAt=7/3 14:30Z`、`MAX(outbound)=7/2 19:06Z`,API 卻回 `2026-07-05 02:00:27`(= 當晚 02:00 摘要 cron 蓋章的 updatedAt);leslie/jenny/uvbookings 同款,值秒級等於各自 updatedAt。

## 2. 根因(診斷)

| 缺陷 | 根因 |
|------|------|
| P0 | `computeLastContactAt`(`adminCustomers.ts:244`)吃到的 `lastOutboundAt` 是 raw `sql<Date>` correlated subquery。drizzle 只解碼「已知欄位」,raw sql fragment 不解碼 → mysql2/TiDB 把 DATETIME 原封當「naive 字串」丟回。舊版直接 `.getTime()`,遇字串就 throw,`rows.map()` 整批爆掉 → 列表空。guestList 走純 SQL 不在 JS 碰 getTime,故只錯日期不會空 —— 兩症狀就此分岔。 |
| P1 | `customerProfiles.updatedAt` 是 `.onUpdateNow()`(`schema.ts:2788`)。02:00 摘要 cron 的 UPDATE 把它蓋成當晚時間;guestList/badge 的 `GREATEST` 用 `COALESCE(..., updatedAt)` 當地板,「最後往來」就塌到 cron 時間。customerList 本來就 fallback 到 `users.createdAt`,沒中這條。本專案「歸檔時間冒充事件時間」**第三次**復發。 |

## 3. 對抗審查另抓到的 blocker(一輪漏修)

四路對抗審查(regression / red-line / adversarial / completeness)在我第一輪修法上又抓到一個 **blocker**:

- guestList 之前把 raw `sql` GREATEST 值**原封送 client**。第一輪只修了 formatter(fix d),但 raw `sql<Date>` 是 naive 無時區字串,client 在 `formatMonthDayLA` 之前先 `new Date(字串)` 用**瀏覽器本機時區** parse,錯日就發生在那一步。結果:非美西 viewer 的 Emerald 顯示 **7/2**;跨 UTC 午夜的 guest 連 Jeff 自己(LA)也錯一天。customerList 沒事是因為它走 `computeLastContactAt` 在 server 端錨定;guestList 沒走。

## 4. 修法(6 檔)

1. **`computeLastContactAt`(customerUnread.ts)**:candidate 一律 coerce 成 Date、parse 不出丟掉、**絕不 throw**;naive DATETIME 字串一律當 **UTC**(`replace(" ","T")+"Z"`),對齊 drizzle 的 timestamp 解碼(`new Date(v+"+0000")`),同一筆記錄 decoded 欄位與 raw 值不再時區打架。→ 修 P0。
2. **guestList**:改成與 customerList **同一支** `computeLastContactAt` 在 server 端算(select 出 inbound / outbound / createdAt 三根指針),錨成 UTC 真 Date,superjson 帶 Date tag 送 client。→ 修 blocker + P1 + 順帶修「createdAt 當 GREATEST 地板會蓋過真 inbound」(純函式是「兩指針皆空才落 createdAt」)。刪掉 select 殘留的 `updatedAt` 死欄位(不讓 cron 時間流進 payload)。
3. **badge(customerUnreadCount)**:orderBy 的 `GREATEST` fallback `updatedAt` → `createdAt`,與 guestList orderBy 逐字一致(top-200 視窗不分岔)。
4. **client `formatMonthDayLA`(adapters.ts,新增純函式)**:`Intl` + `timeZone:"America/Los_Angeles"`,取代 date-fns `format`(吃本機時區);naive 字串亦一律當 UTC(防禦縱深)。useCustomerData 的 formatDate 改用它。→ 修 fix (d)。
5. **註解**:useCustomerData 三處 `updatedAt` → `createdAt`(紅線口徑文件漂移,第三次復發的溫床)。

`updatedAt` 至此從 customerList / guestList / badge / client 渲染的所有「最後往來」口徑徹底消失。
