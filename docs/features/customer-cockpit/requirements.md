# 客戶頁需求書(憲法)

> 2026-07-01 由 Jeff 確認。任何 session 動客戶頁(/ops/customers)前先讀本檔。
> 本檔只寫「要什麼、不要什麼」;「做到哪」看 progress.md,「怎麼做」看各 design.md。

## 一、定位(一句話)

一人公司老闆每天住的那一頁。不是 CRM、不是聊天室、不是後台管理系統,是駕駛艙:
**打開一個客人,五秒看到真相、跑腿已經做完、草稿等著你送、有錯它先攔住你。**

## 二、北極星

讓 AI 把「客人管理的安全跑腿活」自己做掉,不用 Jeff 開口,Jeff 的工作縮到只剩兩件:
**判斷**,跟**按下送給客人**。

心智模型(Jeff 原話):「就好像每一位客人都有自己的 sessions,但經驗是共有的,以及 database。」

## 三、硬邊界(不可違反,violated = P0)

1. **AI 永遠不自動送任何東西給客人。** 信、訊息一律 Jeff 親手送。
2. **AI 永遠不自動產對外報價、不碰錢。** 分界線 = 搬運 vs 生成:搬已驗證售價可以;
   為某客人的日期/人數/必付去「組」一個價 = 客制組價,立刻停、回到 Jeff。
   付款狀態一律 Jeff 手動標。退款走 suggest_action 等 Jeff 審。
3. **準確至上。** 只根據真實資料動作(搬運,不是生成);做不到 100% 正確就停下來等 Jeff。
   AI 出手處顯示的每個日期、金額、名字、交付狀態必須對得上真實來源(Gmail 原串、訂單
   時間戳、DB)。「AI 說做了」必須有工具憑證(tool_result chip),嘴上宣稱不算數。

碰到任何一條:AI 把腿跑完、把東西備好,然後停下來等 Jeff 點頭。

## 四、三條工程原則

1. **大腦做薄**:事實跟對帳全用 code 算(所以永遠對),LLM 只負責把話寫出來。
   時間戳要分清「事件發生時間」vs「我們歸檔時間」;給 Jeff 看的日期一律 America/Los_Angeles。
2. **三色燈**:每個動作出生就分級。綠 = 純跑腿自己做(收信、歸檔、刷新);
   黃 = 備好等 Jeff 一鍵(跟進草稿、建 draft 單);紅 = 送客人/報價/碰錢 Jeff 親手。
3. **看門狗**:拿真資料抓錯,是規則不是判斷。寧可漏報,不可誤報洪水(誤報多了 Jeff 會麻木)。
   兩個數字不齊就不叫,不從別處反推。

## 五、頁面形狀(三欄)

- **左欄**:客人清單(註冊 + 訪客合一),未讀紅點要真的亮,需跟進徽章,搜尋,隱藏/還原。
- **中欄**:真相。header(電話/email)、五秒真相條(換你回/該跟進了/等客人回 + N 天 + 跟進日)、
  ProjectBar 專案 chips(一個信箱多個案子)、四個 tab(概覽/訂單/文件/歷史)。
- **右欄**:Jeff ↔ AI 工作台(不是客人對話紀錄)。問問題、下指令、出草稿、拖檔案。

訪客與註冊客人體驗一致(guest parity)。專案選了 chip,概覽/文件/歷史/chat 全部跟著走。

## 六、操作哲學(從打槍紀錄提煉,視同正式需求)

1. **打字驅動,不要按鈕。** 打「收」就收信、打「跟進日設下週三」就設好、新增客人 = 聚焦
   聊天框打字或拖檔。彈窗表單被刪過一次,不要再蓋回來。
2. **不要 default UI/UX。** 原生 date picker、原生 select 都被拔過。
3. **Chat 是工作台不是聊天紀錄。** Jeff↔AI 歷史保留(不折疊、不混客人信);客人真實往來
   在「最近對話」與歷史 tab。
4. **拖檔案要像 Claude。** 任何檔型丟進聊天框,AI 讀懂(搬運)、建/改單、原檔自動存 R2
   掛進該專案文件 tab,一條龍。純丟檔免打字也能送。不支援要提示,不准靜默。
5. **AI 說話方式**:先查證再開口並講來源、講錯馬上認、分清確定與沒查、先結論後原因、
   有判斷直接給一個建議、話短口語、順手備好下一步。不用「球在誰」這類比喻,說白話。
6. **給客人的文字**(草稿/信件):全程「您」、稱呼延用慣稱、先噓寒問暖再帶事、低壓力
   不催不審問、不破折號、不 markdown、不打勾、純文字、絕不讓客人知道是 AI 寫的、
   不吹牛(沒真的寄出過就不准宣稱已交付)、英文客人給英文稿。
7. **最精簡優先。** 能複用就複用(送信只走既有 escalationReply),不另開平行系統,不過度設計。
8. **prompt 不可靠,紅線要 code 層硬擋。** 破折號、i18n、padding、幻覺成功全是先 prompt
   後失守、最後靠 gate 斷根的案例。新紅線一律直接做 gate。

## 七、非目標(明說不要的)

自動回客人 / 即時 chatbot、自動報價自動定價、任何金流操作、取代 Jeff 判斷、
微信個人號、用 app 管訂單、fine-tune 模型、手機推播/簡訊、排程摘要信、
PDF 全文 RAG 向量檢索、改客人看到的介面、Auto-send 政策卡搬進單客頁。

## 八、驗收標準(scorecard)

拿 Jeff 桌面客人檔案(手工核對過的真相)當考卷,對帳五維度:
1. 正確性(一票否決):金額/名字/已付狀態錯一條 = 不及格
2. 完整性:真實發生的往來/文件/承諾有沒有漏
3. 五秒真相:只看真相條+摘要,五秒答出「哪階段、輪到誰、下一步」
4. 跑腿完成度:信收齊、歸檔好、草稿備好、口氣合格
5. 攔錯:該叫有叫、不該叫沒叫

終極問題:**明天開始只准用這一頁做生意,敢不敢。**
