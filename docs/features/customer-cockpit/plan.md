# 客戶頁駕駛艙 (Customer Cockpit) — 計畫地圖

> 一人公司的客戶頁,做到「打開一個客人,五秒看到真相、跑腿已經做完、草稿等著你送、有錯它先攔住你」。
> 本檔是給 Jeff 跟 AI 的共同地圖。每一步都能單獨上線、單獨見效,照順序走,不用一次扛整個系統。
> 來源:2026-06-24 的 grounded 計畫(現況審計 + 設計 + 安全審查,5 個 agent)。

## 一、目標(北極星)

讓客人 AI 主動把「客人管理的安全跑腿活」自己做掉,不用 Jeff 開口,讓 Jeff 的工作縮到只剩
「判斷」跟「按下送給客人」。

「做完」長這樣:打開任何一個客人,最上面一條就是真相(在哪階段、球在誰、幾天沒回、答應了還沒做的、
下一步),歷史對話自動收齊,跟進草稿用 Jeff 口氣備好等送,任何對不上的(報價對不上成本、名字對不上訂單、
答應沒做)被主動攔下來。

## 二、硬邊界(不可違反)

1. AI 永遠不自動「送任何東西給客人」。信、訊息一律 Jeff 親手送。
2. AI 永遠不自動產對外報價、不碰錢。
3. 準確至上:只根據真實資料動作(搬運,不是生成);做不到 100% 正確就停下來等 Jeff。

碰到上面任何一條,AI 把腿跑完、把東西備好,然後停下來等 Jeff 點頭。

## 三、三條原則(守住就不會越改越亂)

1. 大腦做薄:事實跟對帳全用 code 算(所以永遠對),LLM 只負責把話寫出來。
2. 三條燈號:每個動作出生就分綠/黃/紅。綠=純跑腿自己做、黃=備好等你一鍵、紅=送客人/報價/碰錢你親手。
3. 看門狗:拿真資料幫你抓錯,是規則不是判斷,所以準。

## 四、現況(計畫查證,別重蓋已有的)

- 自動歸檔信件文件:已做好。gmailPipeline 自 2026-06-21 起自動建客人、歸信、把 PDF/文件存進文件 tab。
- 自動收整串歷史:八成。引擎 (customerBackfill) 齊全且 idempotent,只差「新客人出現時自動觸發」的鉤子。
- 自動刷新狀態:九成。每日排程重算 + 手動按鈕 + 開啟時 lazy 刷新都有,只差「有新消息就即時刷」。
- 自動跟進草稿:基礎在(找卡住客人 + 草稿產生器),缺一條把它接起來寫進待審區的鏈。
- 安全審查:四件全部守住硬邊界,自動寄客人預設是關的。可行、安全。

## 五、步驟地圖(照順序,每步可單獨上線)

### Step 0 — 確認地基(驗證,不寫 code)
確認自動歸檔在 prod 真的在跑(信、PDF 有自動歸到對的客人)。你可能已經有比你以為更多的東西。
停在:純驗證。

### Step 1 — 五秒真相條(最高日常價值,零風險)
客戶頁最上面加一條 code 算出來的狀態:階段、球在誰、幾天沒回、未兌現承諾、下一步建議。
全部讀真資料、不碰 LLM、不碰客人。這是你每天每開一個客人都會用到的那塊。
新東西:一個 deterministic `computeCustomerState(profileId)` + 頁面頂部一條渲染。
燈號:綠。Effort:小。

### Step 2 — 自動收歷史(讓真相完整)
新客人一出現就自動把整串 Gmail 收進來,你不用再打「收」。
新東西:gmailPipeline 建檔後 fire-forget 一個 backfill queue job + `threadHistoryCollected` 欄位(防重)+ migration。
停在:只讀只歸,不碰客人、不碰錢。燈號:綠。Effort:小。

### Step 3 — 狀態自動保鮮
有新消息就即時刷新摘要與真相條,不用等隔天排程。
新東西:logInteraction 寫入後 enqueue 一個 summary refresh + Redis debounce(同一人 5 秒內不重排)。
燈號:綠。Effort:小到中。

### Step 4 — 跟進草稿自動備好(黃燈)
偵測到卡住的客人,照真實對話用 Jeff 口氣草擬一封跟進,擺進客戶頁待審區等你一鍵送。
新東西:FollowupAgent + 讀對話的 fetcher + producer 寫進 approvalTasks + 待審卡片 + executor。
停在:絕不自動寄;碰到付款/法律/客訴類跳過不草擬;同一人去重。燈號:黃。Effort:中。

### Step 5 — 看門狗第一條(把「省時間」升級成「省錢」)
先挑一條最會害你賠錢的對帳:客人報價對不對得上後台成本(David 那種漏價)。
踩到就在客戶頁跳一個警示,把兩個數字直接貼給你。規則不是判斷,所以準。
之後再加:護照名對訂單、答應了還沒寄。
新東西:一個小規則引擎 + 客戶頁警示位。燈號:黃(攔你,不替你改)。Effort:中。

## 六、不做的(非目標)

- 自動回客人、自動跟客人對話。
- 自動產對外報價、自動定價。
- 任何金流操作。
- 取代 Jeff 的判斷。

## 七、建議起點

Step 1(五秒真相條)+ Step 2(自動收),一起做。這是你每天最有感、風險最低的一塊,
打開客人立刻不一樣,而且把後面所有步驟的真相基礎打好。
