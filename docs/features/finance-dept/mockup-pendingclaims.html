實作 vs 設計系統：出入清單

忠實還原現網程式，以下是我照實呈現、沒有自作主張改掉的地方。
  1. 原樣呈現財報六分頁的選中態用 bg-teal-600（青綠 #0d9488），不是設計系統的近黑 --primary（已從 teal 改成近黑）。這六個 tab 的配色是既存硬編碼，塊A 只新增「待認領」一個走 i18n 的 tab，沒動既有配色。青綠跟全站黑白極簡有出入，照實呈現。
  2. 建議修候選只顯示訂單號 + 訂金/尾款/全額，沒有客人名字（例：ORD-2026-0087 其實是王先生台灣團）。認領時較難一眼確認是誰的錢，建議候選加上客人名。
  3. 建議修表格沒有「來源帳戶」欄。這三筆都是 Trust 帳戶 Zelle 入帳，但畫面看不出來源；建議加來源欄或列上標 Trust badge，方便對信託合規。
  4. 原樣呈現已自動對帳（auto:exact_amount 等）的入帳不會進這張清單，這張只列 pending。我把自動對帳的例子放表格下方當對比，不是真的在表內。
  5. 原樣呈現沒選候選或分類前，認領鈕是 disabled（灰），對應程式 disabled={!choice}。列 2、列 3 就是這個狀態。
  6. 建議修無分頁：listPending 一次撈 limit 100，存量大時整頁很長，沒有分頁或載入更多。
  7. 建議修無排序 / 無篩選：不能依金額、日期、有無候選排序或篩，筆數一多不好找。
  8. 建議修空狀態很陽春：只有一行「目前沒有待認領的入帳」置中，沒有插圖或引導下一步。
  9. 原樣呈現（我對齊了）PendingClaimsTab 本身沒包 max-w-5xl，現網表格會比上方 switcher 略寬（其他報表有包）。示意圖裡我把它對齊 switcher 寬度方便看；建議程式補上 max-w-5xl 對齊。
