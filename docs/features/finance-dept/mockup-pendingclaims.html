與現有實作 / 設計系統的出入清單（唯讀記錄，未擅自修改版面）

  1. 本頁沒有「已自動比對」清單：PendingClaimsTab.tsx 只呼叫 listPending（只回未認領的入帳），畫面上的「auto:exact_amount」對比卡是本次為了展示對比額外加的說明性元素，ship 後的真實頁面不會出現這張卡片。
  2. 建議修寬度跟上面 tab bar 不一致：ProfitLossV2.tsx 自己有 max-w-5xl mx-auto 包裝，跟上面 tab 切換列同寬對齊；PendingClaimsTab.tsx 沒有這層包裝，表格會撐滿到 main 的全部可用寬度，在寬螢幕上會比上面的 tab bar 明顯更寬，兩者左右邊界對不齊。本 mockup 忠實重現這個落差（沒有偷偷幫它加 max-w），Jeff 可以自己比對上下對不對齊。
  3. 建議修無分頁機制：listPending 查詢寫死 limit: 100，待認領超過 100 筆時沒有翻頁或「載入更多」，UI 上也沒有筆數提示。
  4. 候選/內部分類清單沒有排序或搜尋：表格是純線性清單，無法依日期/金額排序，金額大的舊件跟金額小的新件混在一起。
  5. 建議修空狀態很陽春：只有一行文字置中，沒有圖示或「全部清空，做得好」這類正向回饋，跟本頁其他地方（如損益表的 KPI 卡）比起來略顯單薄。
  6. 顏色 class 不一致：PendingClaimsTab.tsx 直接寫死 Tailwind 灰階（text-gray-900 / bg-gray-50 / border-gray-200），沒有沿用 ProfitLossV2.tsx 已經在用的 text-foreground / border-foreground/10 token 寫法，兩邊視覺上接近但不是同一套 class 來源。
  7. 內部分類只有 4 種固定選項（業主轉帳／利息／Stripe 轉撥／其他），沒有自由文字備註欄位可見於這層 UI（實際 claim mutation 是否接受 note 需另外確認，此頁沒有暴露輸入框）。
  8. 六個分頁只有「待認領」是活的，其餘五個分頁按鈕在本 mockup 中不可點擊。範圍是「待認領」這一頁的預覽，沒有重建其餘五頁內容（但外殼／側欄／兩層 tab bar 全部照真實頁面重建）。
