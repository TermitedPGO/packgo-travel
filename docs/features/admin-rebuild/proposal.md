# PACK&GO 後台系統重建提案

> 2026-06-17 Jeff 拍板：推倒重來，Workspace + Admin 合併成一個系統。

## 問題

現在的後台有三個獨立入口（Workspace、Admin、OpsAgent 頁面），功能分散、半成品多、AI 聽不懂指令。Jeff 每天處理 10+ 個客人，流程全手動：收 email、切到後台查行程、切到桌面 Claude 出報價 PDF、再切回 email 寄出。系統沒有幫到忙。

## 願景

一個系統、兩個入口、AI 是核心。

打開後台，首頁就是 AI 對話框 + 今天該做的事。跟 AI 說一句話就能查資料、執行操作、出報價單。客人頁面一進去就看到完整時間軸，從第一封 email 到出發提醒，不用跳來跳去。

---

## 架構：兩個入口

| 入口 | 內容 |
|------|------|
| **首頁** | OpsAgent 對話框 + Dashboard 卡片（待辦、財務、行程、agent 狀態） |
| **客人** | 客人列表（自動過濾訪客/系統帳號）→ 點進去是客人詳情頁 |

設定（系統清理、agent 設定、供應商設定）藏在右上角齒輪裡，不佔導航。

---

## 首頁：AI + Dashboard

### AI 對話框（OpsAgent）

佔首頁上半部，是整個系統的操作核心。

**能力：**

| 類型 | 範例 | 權限 |
|------|------|------|
| 查資料 | 「David 的訂單狀態」「這個團還有位子嗎」「本月營收」 | 直接做，不用確認 |
| 執行操作 | 「刪掉假資料」「把這個行程下架」「重新抓 UV 行程」 | 先問 Jeff 確認 |
| 產生文件 | 「幫 David 報價日本 7 天」「生成機票確認單」「寫回覆給客人」 | 先問 Jeff 確認 |
| 分析建議 | 「這個團跟競品比怎樣」「客人問這個我該怎麼回」 | 直接回答 |

**技術要求：**

- 接入所有 skill（報價單、機票確認單、簽證、行程比較）
- 接入所有 admin 操作（刪除、上下架、同步、清理）
- 接入客人資料（查詢、時間軸、訂單）
- 接入供應商 API（查餘位、查價格）
- 支援拖 PDF 進對話框（上傳到 R2，AI 讀取內容）

### Dashboard 卡片

對話框下方，四個摘要卡片：

1. **今日待辦** - 未回覆詢問、待確認訂單、待收款、即將出發提醒，按緊急度排序，點擊直接跳到對應客人頁
2. **財務摘要** - 本月營收、待收款總額、Trust vs Operating 餘額
3. **行程總覽** - 上架數量、各區域分布、近期熱門
4. **Agent 狀態** - InquiryAgent 今日處理數、OpsAgent 對話數、供應商同步最後時間、有無失敗

## 客人頁面

### 客人列表

- 只顯示真客人（有 email 且非系統帳號，自動過濾 UptimeRobot 等）
- 搜尋框（名字、email、電話）
- 排序：最近互動優先
- 每列顯示：名字、email、最後互動時間、訂單狀態標籤
- 10+ 客人/天的量級，需要快速切換

### 客人詳情頁

點進客人後，一個頁面看完所有資訊：

**1. 完整時間軸（左側主區）**

一條時間線，從新到舊，所有互動混在一起：
- email 來往（標記方向：來信/回覆）
- 詢問記錄
- 報價單發送
- 訂單建立/付款/確認
- 出發前提醒
- AI 自動回覆記錄

每個事件可展開看細節。

**2. 訂單 + 付款狀態（右側上）**

- 這個客人的所有訂單，每筆顯示：行程名、出發日、人數、總價、已付/待付
- 付款進度條（訂金/尾款）
- 快速操作：確認付款、發送提醒

**3. 檔案區（右側中）**

- 報價單 PDF、護照副本、簽證文件、機票確認單
- 可拖拉上傳
- 按類型分類

**4. 專屬 AI 聊天（右側下或底部）**

- 跟 OpsAgent 討論這個客人的事
- 自動帶入客人 context（「幫他報價」AI 就知道是誰）
- 範例：「幫他查日本 7 天的團」「出一份報價單」「他的簽證進度怎樣」

## 設定（齒輪）

藏在右上角，不常用但必須有：

- 系統清理（刪假資料、purge 供應商行程）
- Agent 設定（自動回覆政策、信任階梯）
- 供應商設定（同步頻率、供應商列表）
- 帳號設定（Gmail 連線狀態、API keys）
