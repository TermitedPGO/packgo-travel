# 提案 — Email 自動回覆(信任階梯式開放)

> Stage 1。起因 2026-06-12 Jeff:「另一個我覺得要計劃的是 email 自動回覆」。
> 背景:同日稍早批9 拍板「全部我核准」並建了 escalation 編輯並回覆;本提案是下一階 —
> **窄類別、可驗證、隨時可關**的真自動回。
> **拍板(2026-06-12)**:① 方向走信任階梯 ✓ ② 永不自動五類(退款/投訴/報價/訂金/簽證)✓
> ③ 達標線 = 20 封 + 95% 不改核准率(Jeff 選更嚴)✓這是在鬆動「永不自動送客人」鐵律,
> 所以整個設計圍繞:先證明、再放行、全留底。

## 現況(2026-06-12 代碼查證,大部分已存在)
- ✅ 政策層:`agentPolicies.inquiry` JSON — autoSendEnabled(false)/ autoSendMinConfidence(85)
  / 每類 action+minConfidence / alwaysEscalate(退款/投訴/critical)
- ✅ 閘門:pipeline meetsAutoSend = 開關 on + 非升級 + 信心 ≥ 門檻
- ✅ 安全網:post-LLM 黑名單(草稿含退款字眼+金額、任何 $ 金額、密碼重設、銀行/卡號
  → 強制升級不寄)— 注入攻擊防線
- ✅ 寄送:sendReplyInThread(回原 thread)+ AGENT_DRY_RUN env 急停
- ✅ 結果記錄:auto_replied / would_auto_send / send_failed 進 interactionOutcomes
- ❌ 缺:影子模式獨立旗標(舊影子靠全域 DRY_RUN,但它現在 false 且被批9 手動回覆共用,
  不能再拿來當影子)
- ❌ 缺:**類別白名單**(現在信心夠就全寄,連報價類都會 — 不可接受)
- ❌ 缺:自動回的可見性(今日待辦留底卡)、數據儀表(核准不改率)、政策開關 UI、日上限

## 目標:三階信任階梯
- **Stage A 影子(預設起點,跑 ~2 週)**:AI 照常出草稿給你核准(現狀不變),但同時
  記錄「這封我本來會自動回」。今日待辦出現淡化資訊卡 + 每週報告:各類別「你不改一字
  直接核准」的比率。**零風險,純收集證據。**
- **Stage B 窄類上線**:某類別影子數據達標(**拍板 2026-06-12:樣本 ≥20 封 + 不改直接核准率 ≥95%**),
  你在政策卡親手開該類別 → 該類高信心信件真自動回。每封自動回 = 今日待辦留底卡
  (可一鍵起草跟進更正信)+ audit + 日上限(預設 10 封)+ 一鍵全停。
- **Stage C 擴類**:看數據逐類重複 B。

## 永不自動(硬編碼,改碼才能動 — 刻意的摩擦)
退款 refund_request、投訴 complaint(既有 alwaysEscalate)+ **報價 quote_request、
訂金 deposit_inquiry、簽證 visa_inquiry**(碰錢/法律,符合「報價求證留人力」原則)。
自動候選從 **general_info**(營業時間/付款方式/一般資訊)起步。

## 非目標
- 不做「全自動客服」:鐵律改為「窄類別白名單自動,其餘照舊全核准」
- 不自動處理帶附件的信(附件 = 通常是證件/文件,人看)
- 不動批9 的 escalation 編輯並回覆(那是人工線,持續存在)

## 成功標準
影子兩週後你能看到一張表:「general_info 23 封,你 21 封沒改直接核准(91%)」→
你開開關 → 之後客人半夜問營業時間 3 分鐘內收到回覆,你早上在今日待辦看到留底卡。
