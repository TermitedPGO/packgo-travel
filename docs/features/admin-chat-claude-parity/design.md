# 整合工作台 — Design (Stage 2 · 定案)

> 來源:proposal.md + 4 路並行設計探索(銷售 / 營運+inbox結構 / 財務 / 行銷+供應商+shell)獨立收斂。
> 一句話:**一個介面,三種檢視(今日待辦 / 全公司事務 / 每個客人),共用導覽 + 時間軸 + 對話 + 勾選。**

## 1. 設計系統(所有卡片繼承)

### 1.1 卡片文法(固定左到右,一堆混排也整齊)
```
[類型 badge] [who chip] [time]                 [☐ 未處理]
body:一句話「這是什麼」
  label/value 欄位(label 灰 w-20,value 黑,數字右對齊成柱)
⚠ warn line(粗黑,不用紅)
[主動作] [次動作]                               [···]
```
- 純黑白、`rounded-xl` 卡 / `rounded-lg` 鈕 / `rounded-md` badge、line icon、等高列、無破折號。
- 類型靠 icon 形狀分,不靠顏色。

### 1.2 狀態 enum(全站唯一一套,學一次到處通)
`未處理` · `等你決定`(要你拍板) · `處理中`(agent 在跑,streaming) · `等外部`(等客人回 / 等供應商 / 等領事館 / 等銀行) · `處理好了` · `錯誤` ; 清單級:`空` · `多筆`。
- 靠形不靠色:左邊條(實心/跑馬虛線/靜止虛線/雙線)+ 角標 pill + 透明度。

### 1.3 勾選(未處理 / 處理好了)
- 每張卡右上固定一顆。勾 → 整卡淡到 ~45% + 收掉動作列 + 下沉到該分組底「已處理」。可還原。
- 這是 Jeff 的**人工處置**;`狀態 chip` 是**系統實況**。兩者可以故意不一致(系統說等客人回,Jeff 還沒勾完)。
- 沒做事就勾「處理好了」→ 軟確認(例:詢問「沒回客人就標好了?」)。

### 1.4 who chip
`@客戶`(頭像首字方塊,黑底白字) 或 `🏢全公司`。在某客人 inbox 內、who = 本人時隱藏(冗餘)。

## 2. 跨域鐵律(每張卡都遵守,寫進元件層不只 UI)

1. **碰錢/碰發布 = 🔒 等你決定,逐筆,禁止整批,禁止自動送**。報價/機票/退款/收費/催款/認列/行銷發布全是 confirm 卡,靜止狀態就是「草稿等你按」。
2. **永不靜默送客人**:所有對客文字(含 AI 草稿、批次行前通知)一律載入「人工過目的 composer」,沒有任何「直接寄出」單鍵。
3. **機票多一層鎖**:卡頂不可關閉的「付款由 Jeff 本人刷卡,系統不輸卡號/CVV、不替你按付款」+ 一個 `待你刷卡` 閒置狀態(agent 停在付款前一步不動)。
4. **spam 不靜默丟客人**:`discard` = 靜音保留 + 「疑似垃圾」匣 + 一鍵「其實是客人→救回」,只有明確「確定是垃圾」才收(仍保留)。匣有計數,永遠看得到。
5. **護照**:加密儲存、全站只顯示 `••••1234`、簽證完成記錄刪除;簽證卡常駐 🔒 安心條。
6. **AI 是輔助不盲信**:信心用人話(「把握 62 分」非裸數字);低信心 + 客訴/退款/緊急一律升 `等你決定`。
7. **trust 永不混淨利**:任何顯示金額處用「雙行 + 金庫 chip + 未認列」三重訊號;`已認列營收` 在上(粗),`客人訂金(信託,未認列)` 在下(細、鎖 icon、縮排),中間一條 hairline,**畫面上永不相加**。
8. **整齊 + 純黑白 + 圓角 + line icon + 無破折號**:急用粗黑非紅;緊急用粗體+pin+badge,不用色。
9. **雙語**:介面 zh-TW 預設 / en;客人原文保留原語言;草稿跟 `preferredLanguage`,一鍵切換。

## 3. Shell(導覽 / 通知 / 搜尋 / 收合)

### 3.1 側邊欄(由上到下)
- **與 AI 對話**(釘最上,唯一一條「自由對話」逃生口,避免硬把客戶當對話串;產出可「升級」成客戶/公司卡)
- **今日待辦**(roll-up,badge = `需要你決定` 件數)
- **全公司事務**(記帳 · 月報 · 行銷 · 供應商,各自 badge)
- **客戶清單**(每個客人 = 獨立 inbox,badge = 該客未處理數,最近未處理優先排序;清完沉底變灰)

### 3.2 今日待辦 = 三桶
1. **需要你決定**(錢/發布/升級/錯誤;絕不隱藏,爆量也全列)
2. **處理中 / 等外部**(在跑,沒卡你)
3. **看一下就好 FYI**(今天 AI 自動處理掉的,可稽核不可刪)
- 每張帶 who chip(唯一客戶+公司混排處)。`空` = 慶祝態「今天沒有需要你決定的事 ✦ AI 已處理 N 件」。跨午夜未處理**順延**標 `逾期`,不消失。
- 排序:① 擋錢/客的錯誤 ② 時效性錢 ③ 一般錢 ④ 發布 ⑤ 升級;同級舊的優先(不爛尾)。

### 3.3 收合 / 行動 / 通知 / 搜尋
- 收合 → 56px icon rail,hover flyout 仍可達;badge 縮成點(錢/錯誤點永遠在)。手機 = 既有 bottom-nav PWA。
- 通知:復用既有 SSE(`opsAgentStream`),錢/錯誤必 toast,FYI 可只 badge;清通知 ≠ 清工作。
- 搜尋:既有 `globalSearch` 擴項目;找客人/行程(含**供應商代碼**一級鍵)/帳/文案;空 query 顯最近聯絡。

### 3.4 客戶 = 取代任意對話串
沒有自由飄的 New Chat 清單;導覽本身就是串列。新對話 = 選一個 context 開打;`+ 新客戶` 建殼錨定陌生人。陌生來信自動建客戶殼(不遺失),可改名/合併。

## 4.5 行銷海報生成(AI / ChatGPT gpt-image)— 多考量

行銷不只文字稿,還要**重生海報**(已接 ChatGPT API / gpt-image)。UX 考量:

- **成本守門(省 API 鐵律)**:每次生成花錢,卡上常駐單價 + 本次累計(例 gpt-image 約 $0.28/張 · 已生成 3 張 ≈ $0.84)。**不自動連發**;「重新生成」是明確一鍵,可設每篇上限。
- **價格不烤進圖**:團名 / 出發日 / 價格用**可編輯的文字疊加層**(template overlay),不烤進 AI 像素。理由:① 烤進去錯了不能改 ② 價格必須對後台,疊加層才能套 §2 的「價格≠後台則擋」檢查。
- **變體選優 + 版本史**:一次出 4 張變體挑一張;生成過的版本留 history 可回選(不必重花錢)。
- **一稿出全平台(核心流程)**:Jeff 丟 flyer 圖 + 廣告文進對話 → 一次生出 6 個平台版本,各自尺寸/語氣/hashtag:**FB**(橫式 1.91:1)· **TikTok**(直式 9:16,影片 → 先出封面 + 短腳本 hook)· **IG**(方形 1:1 + Reels 直式)· **Threads**(文字為主 + 1 圖)· **WeChat 公眾號**(封面 2.35:1 + 長文)· **小紅書**(直式 3:4 種草)。可「只挑幾個平台」或「再生一次」。
- **風格預設**:清新 / 大字報 / 雜誌 / 實景,降低 prompt 負擔。
- **供應商原圖當參考**:可「沿用供應商圖」(標圖權提醒)或「重生」;重生 prompt 自動帶該團真實亮點 + 目的地。
- **生成 ≠ 發布**:出圖後 `採用` / `下載` / `複製文案+圖`,Jeff 自己貼;無自動發布(同 §4 行銷 hard rule)。
- **後端**:`marketingTransformer.ts:127` 現有 TODO「vision 未支援,海報被忽略」就是這塊的缺口,要接 gpt-image 生成 + 疊加層 + 成本計數。
