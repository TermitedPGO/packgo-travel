# Data Fidelity Skill

## 核心原則

此 Skill 定義了所有 AI Agent 必須遵守的資料忠實度規則，確保輸出內容準確、可驗證，且不包含虛構資訊。

## 資料忠實度規則

### 1. 禁止幻覺（Anti-Hallucination）

**絕對禁止**：
- 虛構不存在的景點、飯店、餐廳名稱
- 編造價格、日期、航班號等數字資訊
- 創造虛假的評價、評分或推薦
- 假設未提供的資訊

**正確做法**：
- 只使用原始資料中明確提供的資訊
- 對於缺失的資訊，使用預設值或標記為「待確認」
- 保留原始資料的準確性

### 2. 資料來源追溯

所有輸出資料必須可追溯至原始來源：
- 價格 → 來自網頁的價格標籤
- 日期 → 來自行程表或出發日期列表
- 地點 → 來自行程描述或地圖資訊

### 3. 不確定性處理

當資料不完整或模糊時：
- 使用 `null` 或空字串表示缺失
- 不要「猜測」或「推斷」缺失的值
- 在輸出中標記不確定的欄位

## 驗證清單

在輸出任何資料前，檢查：

- [ ] 所有數字（價格、天數、人數）都來自原始資料
- [ ] 所有名稱（景點、飯店、餐廳）都在原始資料中出現
- [ ] 沒有添加原始資料中不存在的細節
- [ ] 缺失的欄位使用預設值而非虛構值

## 預設值規範

| 欄位類型 | 預設值 |
|---------|--------|
| 價格 | 0 |
| 天數 | 0 |
| 文字描述 | "" |
| 列表 | [] |
| 日期 | null |
| 評分 | null |

## 引用此 Skill

其他 Skill 可以引用此規則：

```yaml
references:
  - data-fidelity
```

這將自動將資料忠實度規則注入到 System Prompt 中。
